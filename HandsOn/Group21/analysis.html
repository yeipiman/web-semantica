DATASET 1 · Speed Cameras (Radars) — City of Madrid

Summary: Inventory of speed cameras (mostly fixed) with location, road/PK, direction, lane and posted speed limit. Useful to provide speed-control context when interpreting accident rates.

Schema (main fields)

Field Type Description
Nº RADAR integer/string Radar identifier.
Ubicacion string Descriptive location (e.g., “M-30, entrance A-1, P.K. 0+500”).
Carretera o vial string Main road or tunnel (e.g., M-30).
PK string Kilometric point (e.g., 0+500).
Sentido string Traffic direction (e.g., inner/outer carriageway).
Tipo string Camera type (Fixed, Section, Traffic-light).
Carril string Affected lane(s) (e.g., “Fixed lane 4”, “All”).
Longitud inicio tramo / Latitud inicio tramo float Start point of a section control (WGS84), if applicable.
X (WGS84) float Easting in meters (UTM/ETRS89 Zone 30 by range).
Y (WGS84) float Northing in meters (UTM/ETRS89 Zone 30 by range).
Longitud float Longitude (WGS84, decimal degrees).
Latitud float Latitude (WGS84, decimal degrees).
Velocidad límite integer Speed limit at location (km/h).
Responsable string Responsible body (e.g., DGP / City Council).

Quantities & Value Ranges

Volume: several dozen cameras across the city.

Coordinates (Madrid expected): WGS84 lat ≈ 40.33–40.52; lon ≈ −3.78 to −3.57. UTM/ETRS89 (EPSG:25830) X ≈ 438,000–456,000; Y ≈ 4,442,000–4,505,000 (meters).

Speed limit: 30–90 km/h.

DATASET 2 · Population by District and Neighbourhood (Annual)

Summary: Annual resident population for Madrid’s administrative units. Used to normalize accident counts into rates per 100,000 inhabitants.

Schema (main fields)

Field Type Description
fecha date/string Reference date (e.g., “1 de enero de 2024”).
cod_municipio string Municipality code (Madrid = 28079).
municipio string Municipality name.
cod_distrito integer District code (1–21).
distrito string District name.
cod_barrio integer Neighbourhood code (within its district).
barrio string Neighbourhood name.
num_personas integer Total residents.
num_personas_hombres integer Male residents.
num_personas_mujeres integer Female residents.

Quantities & Value Ranges

Coverage: ≈131 neighbourhoods × years; 21 districts × years.

Population: integer ≥ 0; from thousands (neighbourhoods) to hundreds of thousands (districts).

DATASET 3 · Traffic Accidents — City of Madrid (Per Involved Person)

Summary: Microdata per person involved in each accident: date/time, location, district, contributing factors, and injury severity.

Schema (main fields)

Field Type Description
num_expediente string Accident case ID (shared by all persons in the same crash).
fecha date Date of accident (DD/MM/YYYY).
hora time Time of accident (H:MM:SS).
localizacion string Intersection or street segment.
numero string/integer Street number / position.
cod_distrito integer District code.
distrito string District name.
tipo_accidente string Collision type (e.g., head-on, side, run-over).
estado_meteorológico string Weather (clear, rain, fog…).
tipo_vehiculo string Vehicle class (car, moped, motorcycle, VMP…).
tipo_persona string Role (driver, passenger, pedestrian…).
rango_edad string Age band (e.g., “30–34”).
sexo string Sex.
cod_lesividad integer Injury code.
lesividad string Injury description (e.g., “No medical care”, “Serious injury”, “Fatality”).
coordenada_x_utm float Easting (UTM/ETRS89 Zone 30, meters).
coordenada_y_utm float Northing (UTM/ETRS89 Zone 30, meters).
positiva_alcohol string Alcohol positive test (Y/N).
positiva_droga string Drug positive test (Y/N).

Quantities & Value Ranges

Volume: tens of thousands of rows per year (persons). Time range 00:00–23:59.

Coordinates (UTM/ETRS89 Z30): X ≈ 438,000–456,000; Y ≈ 4,442,000–4,505,000 (meters).

Severity: coded (e.g., 0–14) plus textual labels.

Minimal Integration Across Datasets

  • Rate per 100,000 inhabitants (year × district/neighbourhood): rate = (accidents or victims / population) × 100,000.
  • Speed-control context: compare rates in areas with vs without a speed camera; if installation date exists, run a before/after comparison.
  • Keys: temporal join by year (derived from fecha), spatial join by cod_distrito / distrito (and cod_barrio / barrio where used). Optionally use geometry to compute distance from accident to nearest camera.

Ranges & Units (Summary)

  • Speed limit (radars): 30–90 km/h.
  • Population: persons (integer ≥ 0).
  • Accidents: counts (event or involved person).
  • Coordinates: UTM/ETRS89 Zone 30 (meters) and WGS84 (decimal degrees) where provided.

Resource Naming Strategy

Domain:

https://www.roadsafetydata.com/

The URIs are divided into two types:

  • URIs for defining the ontology
  • URIs for defining the initialized resources

Ontology URIs

All custom ontology URIs begin with: https://www.roadsafetydata.com/ontology#

Example classes:

  • Accident
  • Radar
  • District
  • Population

Example object properties:

  • hasRadar
  • takesPlaceIn
  • hasPopulation

Example datatype properties:

  • hasAccidentDate
  • hasVehicleType
  • hasSeverity
  • hasPopulationCount

Resource URIs

Resource URIs should follow the format:

  • General Format: https://www.roadsafetydata.com/resource/Accident/{num_expedient}
  • Specific Example: https://www.roadsafetydata.com/resource/Accident/2025S000056
  • Other Examples:
    • https://www.roadsafetydata.com/resource/Radar/M-30_EntradA1
    • https://www.roadsafetydata.com/resource/District/Chamartín
