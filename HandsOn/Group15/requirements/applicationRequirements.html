Application Requirements — “Madrid Air Now”

Aplicación web para visualizar y explorar la calidad del aire por estación/barrio/fecha, con gráficos y mapa.

Functional requirements

  • F1. Selector de fecha/intervalo y estación/barrio.
  • F2. Mapa con marcadores de estaciones (color por nivel NO2/PM10).
  • F3. Serie temporal por estación y magnitud.
  • F4. Tabla descargable (CSV) del subconjunto filtrado.
  • F5. Enlaces externos (Wikidata/DBpedia) para cada estación/localización.

Non-functional requirements

  • N1. Datos abiertos (licencia compatible).
  • N2. Rendimiento: carga inicial < 3s con muestra local / paginación para histórico.
  • N3. Accesibilidad AA y diseño responsive.

Data & linking plan

  • CSV ➜ RDF (Turtle/JSON-LD) con vocabularios QUDT/SSN/Geo.
  • Identificadores estables para estaciones; sameAs a entidades externas cuando existan.
  • Posible enriquecimiento con meteorología (temperatura, viento).

Acceptance criteria

  • AC1. El usuario puede filtrar por fecha y estación y ver el mapa coloreado.
  • AC2. Se muestra la serie temporal y se puede descargar CSV filtrado.
  • AC3. Cada estación tiene enlaces “Más info” a fuentes externas.
