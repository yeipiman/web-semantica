Dataset: Información de estaciones – Red de calidad del aire (Madrid)
Source / Portal: Ayuntamiento de Madrid – “Calidad del aire. Estaciones de control”

R1. Smart City Domain [Required]

Environment / Air Quality. The dataset lists air-quality monitoring stations, their locations and identifiers, which fits the environment domain.

R3. Open License [Required]

Madrid Open Data applies the city’s general reuse conditions, which permit reuse for commercial and non‑commercial purposes, including copying, diffusion, modification and combination of the information, with the duty to cite the source and not imply sponsorship, among other conditions.

R4. Linkability to Real‑World Entities [Required]

Fields supporting linkage to other datasets:

  • Coordinates: LONGITUD_ETRS89, LATITUD_ETRS89, COORDENADA_X_ETRS89, COORDENADA_Y_ETRS89, LONGITUD, LATITUD
  • Stable IDs: CODIGO, CODIGO_CORTO, COD_TIPO, COD_VIA
  • Addresses: DIRECCION, COD_VIA, VIA_CLASE, VIA_PAR, VIA_NOMBRE
  • Temporal: Fecha alta

These enable joins with OSM/IGN layers, administrative divisions, and weather series (by coordinates and date/time).

R5. Documentation [Optional]

  • Associated document (data interpretation / structure): available from the stations page as “Documento de estructura Estaciones de Control (PDF)”.
  • For real‑time observations, the portal’s “Datos en tiempo real” page explains the refresh cadence (approx. every 20 minutes at :15, :35, :55) and links a detailed PDF for interpretation.
  • See also: Calidad del aire – Datos en tiempo real.

R6. Multiple Sources [Optional]

Not multi‑source. Based on the official dataset pages, this dataset does not appear to integrate multiple external sources. The Responsible unit is Dirección General de Sostenibilidad y Control Ambiental and the Editor is Ayuntamiento de Madrid, indicating a single institutional source (see the stations dataset page). The real‑time page indicates the values are automatically produced by the monitoring stations, with no mention of external aggregation (see real‑time dataset page). Therefore, R6 is not met (discussed). If multi‑source integration is desired for extra credit, consider joining with external datasets (e.g., meteorological series) using station codes and/or coordinates plus timestamps.

Known Issues / Risks

  • Specify/validate coordinate reference systems (ETRS89 vs WGS84) when linking with external GIS layers.
  • Check for nulls and consistency across identifiers/addresses.
  • Comply with attribution under the Madrid Open Data reuse conditions when publishing.
