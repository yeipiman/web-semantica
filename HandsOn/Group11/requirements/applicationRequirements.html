EventFinder Madrid - Application Requirements

Proposed Application: EventFinder Madrid

Web and mobile application integrating cultural, sports, and innovation events data from Madrid to (i) explore upcoming events, (ii) find activities by area and type, and (iii) discover events based on user preferences and location.

Users and Goals

  • Citizens: Find nearby events and activities matching their interests
  • Event organizers: Reach target audiences effectively
  • City administration: Monitor event distribution and cultural engagement

Functional Requirements (FR)

  • 1. Interactive map with event locations and filtering capabilities
  • 2. Advanced search with filters by date, time, type, and district
  • 3. Detailed event information panel (description, time, location, capacity)
  • 4. Personalized event recommendations based on user preferences
  • 5. Export and share event information

Non-Functional Requirements (NFR)

  • 1. Data and metadata will be published as RDF using vocabularies: schema.org (events, places), Time Ontology, DCAT (catalog)
  • 2. External links: Madrid districts (open cartography), Wikidata for venues and locations
  • 3. Original data licenses preserved in metadata (dcterms:license)
  • 4. WCAG 2.1 AA accessibility compliance and public SPARQL endpoint
