Hands-On 2: Analysis

1. Dataset Analysis

The source file is a semicolon-separated CSV (;) with 56 columns and 126 rows.
Columns: PROVINCIA, MUNICIPIO, ESTACION, MAGNITUD, PUNTO_MUESTREO, ANO, MES, DIA, H01–H24, V01–V24.

  • PROVINCIA: Province code (constant 28 = Madrid)
  • MUNICIPIO: Municipality code (constant 79 = Madrid city)
  • ESTACION: Station code (23 possible values)
  • MAGNITUD: Pollutant or measured magnitude code (11 different types)
  • PUNTO_MUESTREO: Sampling point identifier (e.g., 28079011_12_8)
  • ANO/MES/DIA: Measurement date (e.g., 2025/10/05)
  • H01–H24: Hourly numeric concentration values
  • V01–V24: Validation codes (V = valid, N = null/unavailable)

Value ranges: Hourly concentration values range from 0.0 to 102.0. Validation codes are “V” or “N”.

3. Resource Naming Strategy

  • Ontology: http://madrid-airquality.es/ontology#
  • Station: http://madrid-airquality.es/station/{station}
  • Measurement: http://madrid-airquality.es/measurement/{sampling_point}/{datetime}
  • Pollutant: http://madrid-airquality.es/pollutant/{magnitude}

Technical Notes

  • The CSV file only contains data for the current day, the dataset is updated in real time. A new file is generated each day with the corresponding daily measurements.
  • Hours marked as “N” correspond to time slots that have not yet been measured or validated at the time of download.
