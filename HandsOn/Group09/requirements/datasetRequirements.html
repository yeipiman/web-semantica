Dataset Requirements Analysis

This document analyzes whether the selected datasets for the AirViz Madrid application satisfy the defined requirements (R1–R6).

Mandatory Requirements (R1–R4)

Requirement Description Dataset Satisfaction
R1 Data are in one of the selected smart city domains. [x] The selected datasets belong to the air quality and urban monitoring domain, which is part of the smart city domains.
R2 Data are available as a CSV file. [x] All datasets are provided in CSV format, ensuring ease of use for processing and visualization.
R3 Data have an open license so they can be published. [x] The datasets are released under open licenses (e.g., CC-BY or public domain), allowing redistribution and publication in the AirViz Madrid platform.
R4 Data can be easily linked with generic real-world entities (e.g., locations). [x] The datasets include geographic coordinates (latitude and longitude) and standardized identifiers, making it straightforward to link them with real-world locations in Madrid.

Optional Requirements (R5–R6)

Requirement Description Degree of Satisfaction
R5 Documentation exists for the data. Partial: Some datasets provide detailed documentation including variable descriptions and data collection methods. Others have minimal metadata, requiring additional exploration to fully understand the content.
R6 Data can come from multiple data sources. Partial: While the core dataset comes from a single open data portal (Madrid City Council), additional external datasets (e.g., meteorological data) could be linked. Integration across multiple sources is possible but not fully implemented in the current selection.

Conclusion

The selected datasets fully satisfy all mandatory requirements (R1–R4), making them suitable for the AirViz Madrid application. The optional requirements (R5–R6) are only partially satisfied, but with minor additional effort, full compliance could be achieved, especially in terms of documentation and multi-source integration.
