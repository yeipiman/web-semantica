Dataset Analysis

1. Dataset Analysis

This section provides an overview of the selected dataset. It summarizes the initial exploration performed using OpenRefine.

  • Dataset title: Air quality. Real-time data
  • Source: data.europa.eu – Ayuntamiento de Madrid
  • Format: CSV
  • Number of rows: 126
  • Number of columns: 56

The dataset provides hourly air quality measurements collected across multiple monitoring stations in Madrid. Each record represents a specific pollutant measured at a given location and time, along with a corresponding validation flag. This dataset was analyzed following the official documentation of Madrid's open air quality data.

Main columns include:

  • PROVINCIA, MUNICIPIO, ESTACION → identify the measurement location (province code, municipality code, station code. See official documentation, Annex I, for the mapping code → station name).
  • MAGNITUD → pollutant code (see official documentation, Annex II, for the mapping code → pollutant name).
  • PUNTO_MUESTREO → complete station code (province + municipality + station code followed by the pollutant code and the sampling technique code. See official documentation, Annex II, for the mapping code → sampling technique)
  • ANO, MES, DIA → date of the measurement (year, month, day).
  • H01–H24 → hourly measurement values for the given pollutant (hours 01 to 24).
  • V01–V24 → validation flags corresponding to each hourly value.

The dataset includes real-time measurements for 11 different pollutants (magnitudes 1, 6, 7, 8, 9, 10, 12, 14, 20, 30, and 35) across 23 different monitoring stations in Madrid (all of the active ones except for the one located in Urb. Embajada (Barajas)). Each pollutant value is recorded hourly (columns H01–H24, H24 representing the last hour of the day), with a corresponding validation flag (V01–V24). All the values are measured within the city of Madrid (columns province code 28 and municipality code 079). Explanation for missing values is given in the last page of the official documentation.

The dataset covers 126 records, representing one full day of measurements (October 9th, 2025). Most hourly values range between 0 and 100 for pollutants. Missing or invalid values are denoted by N in the validation columns (V01–V24).

3. Resource Naming Strategy

The resource naming strategy defines how URIs will be structured for both the ontology and the generated RDF data.

Base URI

All resources will be published under the following base namespace: 

http://example.org/madrid-air-quality/

Namespaces

  • Ontology: http://example.org/madrid-air-quality/ontology/
  • Data: http://example.org/madrid-air-quality/data/

URI Patterns

  • Stations: http://example.org/madrid-air-quality/data/station/{stationID}
  • Pollutants: http://example.org/madrid-air-quality/data/pollutant/{pollutantCode}
  • Measurements: http://example.org/madrid-air-quality/data/measurement/{puntoMuestreoID}/{hour}
  • Sampling points: http://example.org/madrid-air-quality/data/sampling-point/{puntoMuestreoID}
  • Ontology terms: http://example.org/madrid-air-quality/ontology/{term}

Content Negotiation

Each resource URI supports content negotiation. When accessed through a web browser, the HTML representation will be returned. When accessed by semantic web agents (using appropriate HTTP headers), RDF representations such as Turtle or JSON-LD will be provided.
