1. Analysis of the Datasets

Dataset 1: Arbolado en parques y zonas verdes de Madrid

Source: Ayuntamiento de Madrid, datos.madrid.es
URL: Arbolado en parques y zonas verdes

AttributeDescriptionExample / Value Range
NOMBRE PARQUEName of the park or green area.jardines del buen retiro, parque madrid rio, etc.
ESPECIESpecies of the tree (scientific name).Aesculus hippocastanum, Platanus hispanica, etc.
UNIDADES 2024Number of trees of this species in the park (for year 2024).Integer values, between 1 and 6200.

Data characteristics: The dataset contains around 308 records, each representing the number of trees of a specific species in a given park. Values are integers, and textual fields such as park names and species names are complete and consistent. The data is linkable: parks can be connected to geospatial datasets, and species names can be linked to external resources such as Wikidata or DBpedia.

Data characteristics: The dataset includes 1,632 municipal green zones, one record per area. Each record contains the district, main category (apartado), urban nature (naturaleza del inmueble), location (situación destino), intended use (destino), and area in square meters (solar). Values are mostly textual, except for the area. The data is linkable: districts and green zone types can be mapped to geospatial vocabularies or external linked datasets, and textual descriptions enable alignment with municipal and open data resources.

3. Resource Naming Strategy

All resources will be identified using stable HTTP URIs, following a simple slash-based pattern. URIs will provide both human-readable (HTML) and machine-readable (RDF) representations using content negotiation.

Base namespace:
https://example.org/group06/

URI patterns:

  • Ontology terms: https://example.org/group06/def/{Term}
  • Municipal green zones: https://example.org/group06/zone/{zoneId}
  • Trees in parks: https://example.org/group06/tree/{treeId}
  • Districts: https://example.org/group06/district/{districtName}

For example, a tree in Parque del Retiro could have the URI: https://example.org/group06/tree/12345, and the corresponding green zone: https://example.org/group06/zone/678.

These URIs are designed to be permanent. If a resource moves, HTTP redirects will maintain accessibility and avoid breaking links.

4. Ontology design

We provide a lightweight ontology tailored to our datasets, reusing standard vocabularies:

  • Reuse: schema:address
  • Core classes: ex:GreenZone, ex:Tree, ex:District, ex:Species
  • Object properties: ex:locatedIn, ex:hasTree, ex:hasSpecies
  • Datatype properties: ex:zoneId, ex:parkName, ex:category, ex:nature, ex:intendedUse, ex:area, ex:treeCount
