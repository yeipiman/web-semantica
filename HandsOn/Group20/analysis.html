Aparcamientos EMT — Data Source Analysis, Licensing & Resource Naming Strategy

Dataset analysed: parkings.csv (EMT Madrid Open Data). This document summarises the detailed analysis of the data source, its licensing aspects, and the resource naming strategy defined for the dataset to be generated.

Data Source Summary
  • Source: EMT Madrid Open Data — Aparcamientos EMT (https://datos.emtmadrid.es/dataset/aparcamientos-emt)
  • Format: CSV, GeoJSON, KML, SHP
  • Geolocation: Latitude/longitude provided for each record
  • Domain: Smart Mobility / Parking
  • License: Creative Commons Attribution 4.0 (CC-BY 4.0)

1. Data Source Analysis

The Aparcamientos EMT dataset contains information about public and private parking facilities in Madrid managed by EMT. Each record provides metadata such as the facility name, address, coordinates, category, and type (rotation, resident, etc.). The dataset was analysed to assess structure, quality, and potential for integration.

1.1 Quantitative Overview

  • Rows: 90
  • Columns: 17 (id, family, name, address, areaCode, zrea, category, catCode, country, email, URLIcono, long, lat, state, town, emtParking, type)
  • Coordinates: Complete (100% of rows contain valid lat and long values)
  • Missing values: areaCode ≈ 59%, email ≈ 82%
  • Spatial coverage: All records located within the city of Madrid (lat 40.36–40.49; long –3.78––3.60)

1.3 Relationships and Data Quality

  • Each parking lot (Parking) belongs to one city (Madrid).
  • Administrative areas (districts) can be inferred from coordinates.
  • Addresses are in free-text format; normalization recommended for future integration.
  • All coordinate pairs are valid; ready for use in map-based applications.
  • Potential additional fields (capacity, hours) could be added later to enrich the model.

2. Licensing Analysis

The dataset is distributed under the Creative Commons Attribution 4.0 International (CC-BY 4.0) license. This license allows reuse, modification, and redistribution provided that attribution is given to the original source (EMT Madrid).

  • Allowed: Share, adapt, and combine the data for any purpose, including commercial use.
  • Requirements: Attribution to EMT Madrid and indication of modifications if any.
  • Recommended Attribution:
    “Data sourced from EMT Madrid Open Data (Aparcamientos EMT dataset), licensed under CC-BY 4.0.”

Proposed License for Derived Dataset: The derived dataset, which may integrate EMT data with other open resources, should also adopt the CC-BY 4.0 license to preserve openness and compatibility.

3. Resource Naming Strategy

This strategy ensures that datasets, entities, and versions are uniquely and consistently identified through permanent and dereferenceable URIs, supporting linked data principles.

Resource TypeNaming PatternExampleNotes
Dataset (overall) https://data.smartcitymadrid.es/dataset/parking-emt https://data.smartcitymadrid.es/dataset/parking-emt Main entry for the dataset collection.
Parking resource (individual) https://data.smartcitymadrid.es/id/parking/{id} https://data.smartcitymadrid.es/id/parking/arguelles Each parking facility gets a unique identifier derived from its id or normalized name.
Dataset version https://data.smartcitymadrid.es/dataset/parking-emt/{YYYY-MM-DD} https://data.smartcitymadrid.es/dataset/parking-emt/2025-10-05 Facilitates dataset versioning and reproducibility.
API endpoint https://api.smartcitymadrid.es/parking?type={type}&lat={lat}&long={long} https://api.smartcitymadrid.es/parking?type=rotation&lat=40.42&long=-3.70 For live queries by filters and coordinates.
