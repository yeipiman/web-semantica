1. Application Overview

The application will leverage the Aparcamientos EMT dataset to allow users to search for and locate parking lots in Madrid. Users will be able to filter by type of parking lot (e.g., rotation, residents, park-and-ride), view results on a map or list, and explore parking details.

2. Functional Requirements

  • Search by type: Users can select parking lot categories (rotation, residents, park-and-ride, etc.).
  • Search by location: Users can enter an address or use geolocation to find nearby parking lots.
  • Filter options: Filters for availability of residents-only, rotation, 24h, electric charging stations, etc.
  • Map view: Display parking lots on a map with markers based on coordinates (lat,long).
  • List view: Display parking lots in a scrollable list with name, address, type, and distance.
  • Detail view: Each parking lot shows address, type, capacity, opening hours, and a link to EMT page.
  • Favorites: Users can save favorite parking lots for quick access.
  • Attribution: The app must show attribution to EMT open data (CC‚ÄëBY license).

3. Non-Functional Requirements

  • Performance: Fast search and filtering (under 2s for results).
  • Usability: Clear interface, mobile-friendly layout, accessible color contrast.
  • Scalability: Ability to add additional datasets (traffic, occupancy sensors) in the future.
  • Internationalization: Support for Spanish (default) and English.

4. User Interface Description

The user interface will have three main sections:

  1. Home/Search Screen: Search bar, type filter dropdown, location input, and quick access buttons.
  2. Results Screen: Map view with pins for parking lots and a toggle for list view.
  3. Detail Screen: Displays details of the selected parking lot with options to save as favorite.
