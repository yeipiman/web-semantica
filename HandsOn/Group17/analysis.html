Data Analysis

1. General Description

The dataset contains the names of various parks, gardens and green zones and the name of trees in them. There are 3 columns, the first column has the name of the parks or green zones, the second one has the names of the trees and the third one has the amount of trees in each park or green zone. A more detailed description below:

  • Number of lines with relevant data (parks, tree names, etc.): 274
  • Number of lines without relevant data: 34
  • Number of columns: 3
  • Number of different parks, gardens and green zones: 24
  • Number of different tree species: 103
  • Range of values in "Cantidad 2024" column: 1 (min) – 6200 (max)

2. Data Schema

Column Description Data Type
NOMBRE PARQUE Name of the park, garden, or green zone Text
ESPECIE Scientific name of the tree species Text
UNIDADES 2024 Number of trees of that species in a specific park Integer

3. Data issues

  • 34 rows are empty or do not contain relevant data.

5. Resource Naming Strategy

For the generated Linked Data, each resource will have a unique URI following these conventions:

  • URI form: Slash URIs (/) are used for everything.
  • Domain: https://group17.project/
  • Base path for resources: /madrid-parks/
  • Prefix: web="https://group17.project/madrid-parks"
  • Park resources: web/park/ + park name (e.g., https://group17.project/madrid-parks/park/Retiro)
  • Tree species resources: web/tree/ + scientific name (e.g., https://group17.project/madrid-parks/tree/Quercus_robur)
  • Ontology classes and properties: web/ontology/ + class or property name (e.g., https://group17.project/madrid-parks/ontology/hasTree)
  • Content negotiation: URIs can be prepared to serve both HTML and RDF representations.
